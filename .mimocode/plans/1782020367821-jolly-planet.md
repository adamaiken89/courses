# Plan: Nested Heading Indentation in Reader Sidebar

## Problem

`ReaderView` sidebar renders all sections flat — `##` and `###` headings appear at the same indent level. `ModuleSection.level` is parsed but unused visually.

## Current State

- `ModuleSection` (ModuleSection.swift:3-27) — parses `##` (level 2) and `###` (level 3) into flat list
- `ReaderView.sectionList()` (ReaderView.swift:108-137) — renders all sections with identical `padding(.leading, sectionIndent)`, no level differentiation
- Lessons use `##` as major sections, `###` as subsections (confirmed in sample lesson.md files)

## Design: Indentation + Visual Hierarchy

### Changes

**1. `ReaderView.swift` — `sectionList()` and `moduleRow()`**

Indent `###` sections under their parent `##`:
- `##` (level 2): current indent, normal font, accent bullet
- `###` (level 3): `+DesignConstants.Padding.sectionSubIndent` extra indent, `.caption` font, smaller/lighter bullet

Specifically in `sectionList`:
```
.padding(.leading, section.level == 3 
  ? DesignConstants.Padding.sectionIndent + DesignConstants.Padding.sectionSubIndent 
  : DesignConstants.Padding.sectionIndent)
```

Sub-section bullets: reduce `sectionBulletHeight` by ~2pt, use `AppColors.secondaryLabel` instead of accent for the vertical line.

Sub-section heading text: use `.caption` font (smaller than current `.caption` for level 2), lighter color for level 3.

**2. `DesignConstants.swift` — Add indent constant**

```swift
enum Padding {
  static let sectionSubIndent: CGFloat = 16  // extra indent for ### under ##
}
```

**3. `ModuleSection.swift` — Expand level support (optional stretch)**

Add `####` (level 4) parsing to support deeper nesting if present. Low priority — current lessons only use h2/h3.

### Visual Result

```
Module 1: React 19 Architecture Overview
├ Learning Objectives          ← ## (level 2, normal indent)
├ Core Content                 ← ## (level 2, normal indent)
│   SDL: Schema Definition     ← ### (level 3, indented)
│   Scalar Types               ← ### (level 3, indented)
│   Object Types               ← ### (level 3, indented)
├ Summary                      ← ## (level 2, normal indent)
```
